mini-inet — a simulated exchange built from first principles

Stage 1 — live Order flow through a matching engine
Limit order book + matching engine with price-time priority and price-improvement-to-aggressor. Visualised: client → TCP Port → ME → multicast fan-out to MD, Drop, Trade Reporter, and a passive standby ME.
Open the Stage 1 visual →
Stage 2 — planned Sequenced event stream + state-machine replication
Persisted log of ME events. Consumers replay from seq 0 to rebuild state. "Shoot any one in the head" — restart, replay, identical state.
Stage 3 — planned Per-topic sequence numbers + the cancel-vs-execution race
Optimistic concurrency with one transaction in flight per topic. Reproduce the "bow-tie" diagram: ME silently drops a stale-seq cancel; the client emits a sequenced cancel-reject from its own next seq.
Stage 4 — planned Ports, retransmitter, multicast bus, passive ME failover
Real TCP ports, an RT that records the stream and answers gap-fills, UDP multicast (or unicast-fanout fallback in cloud), and a passive ME that listens to the primary's output and can fail over.
Stage 5 — planned Cancel/Auction fairies + latency measurement
Offloaded stream consumers for cancels and auctions. Tick-to-trade latency measured with HdrHistogram (open-loop, no coordinated omission). Find the cache-miss hot path.
